EDGE-NATIVE SCALABILITY SUMMARY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. BROWSER-DRIVEN GENERATION & SENDING LOOPS ✅
   - When a user clicks "Generate", the frontend calls Google APIs directly
     and uploads the generated PDF directly to Cloudflare R2 via presigned URLs.
   - The API Worker never handles raw PDF buffers or long-running HTTP connections,
     eliminating OOM crashes and socket exhaustion completely.
   - When sending certificates, the frontend iterates and sends asynchronous,
     parallelized requests to the `/send` or `/send-whatsapp` endpoints.
   - Heavy Redis/BullMQ queues are completely eliminated.

2. CLOUDFLARE D1 AT THE EDGE ✅
   - Replaced PostgreSQL / Firestore databases with Cloudflare D1 (SQLite at the edge).
   - High-speed relational queries run in the same edge data centers as your Worker instances.
   - No connection pool limits.

3. COST INFRASTRUCTURE ✅
   - Replaced $18/mo DigitalOcean Droplets and $25/mo Supabase Pro with Cloudflare's free/low-cost tiers.
   - R2 storage features zero egress fees.

CURRENT SYSTEM AWARDS & ARCHITECTURE TALKING POINTS

- Migrated legacy Express.js/PostgreSQL to Cloudflare Workers (Hono) and Cloudflare D1 (SQLite).
- Client-side certificate generation utilizing user's individual Google API quotas.
- Public certificate verification page with dynamic QR codes.
- WhatsApp Interactive list bot + Telegram SUPERGROUP Support Bridge:
  - Meta Webhooks hit the Cloudflare Worker.
  - Conversational states are managed in D1 (`interactions`, `user_states`).
  - Active support conversations are bridged to Telegram topics (threads) under a single Supergroup.
- Payment gateway (Cashfree) with idempotent ledger updates based on order IDs in the D1 ledger.
- Codegen: OpenAPI Spec (api-spec) dynamically generates `api-zod` and `api-client-react` packages.
